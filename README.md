# FastAPI Support Tickets

REST API для тикетов службы поддержки на FastAPI + MySQL.

## Возможности

- Создание тикетов с прикреплением файлов (изображения и видео)
- Комментарии к тикетам с файлами
- Фильтрация тикетов по дате, статусу и owner
- Управление владельцами (owners) через административный API
- Сообщения (messages) между системами
- UUID для безопасного доступа
- Два типа API-ключей: пользовательский и административный
- Поддержка CORS и i18n (русский / английский)

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env
# Отредактируйте .env — укажите параметры БД и API-ключи
uvicorn main:app --reload
```

## Переменные окружения (.env)

| Переменная      | Описание                          |
|-----------------|-----------------------------------|
| `DATABASE_URL`  | URL подключения к MySQL           |
| `API_KEY_USER`  | API-ключ для пользователей        |
| `API_KEY_ADMIN` | API-ключ для администраторов      |
| `UPLOAD_DIR`    | Папка для загружаемых файлов      |

Создать API ключ:
```bash
python -c "import uuid; print(uuid.uuid4())"
```

## Аутентификация

Передавайте ключ в заголовке `X-API-Key`.

- `API_KEY_USER` — доступ к пользовательским эндпоинтам
- `API_KEY_ADMIN` — доступ ко всем эндпоинтам, включая административные

## API эндпоинты

### Тикеты — пользовательские (API_KEY_USER или API_KEY_ADMIN)

| Метод | URL                              | Тело / параметры                                                        | Описание                       |
|-------|----------------------------------|-------------------------------------------------------------------------|--------------------------------|
| POST  | `/tickets`                       | multipart: `owner_uuid`, `subject`, `name`, `email`, `message`, `file?` | Создать тикет                  |
| GET   | `/tickets/{uuid}`                | —                                                                       | Получить тикет с комментариями |
| POST  | `/tickets/{uuid}/comments`       | multipart: `text`, `author?`, `file?`                                   | Добавить комментарий           |
| PATCH | `/tickets/{uuid}/close`          | —                                                                       | Завершить тикет (статус closed)|

### Тикеты — административные (только API_KEY_ADMIN)

| Метод | URL                              | Тело / параметры                                                                  | Описание                        |
|-------|----------------------------------|-----------------------------------------------------------------------------------|---------------------------------|
| GET   | `/admin/tickets`                 | query: `access_key` (обяз.), `page`, `page_size`, `status`, `date_from`, `date_to` | Список тикетов owner'а         |
| GET   | `/admin/tickets/{uuid}`          | —                                                                                 | Тикет с комментариями           |
| PATCH | `/admin/tickets/{uuid}/status`   | JSON: `{ "status": "..." }`                                                       | Изменить статус тикета          |

> `GET /admin/tickets` аутентифицируется через query-параметр `access_key` (ключ владельца), а не через заголовок `X-API-Key`.

### Владельцы (owners)

| Метод  | URL                               | Доступ    | Тело / параметры                        | Описание                       |
|--------|-----------------------------------|-----------|-----------------------------------------|--------------------------------|
| POST   | `/admin/owners`                   | Admin     | JSON: `{ "name", "description" }`       | Создать владельца              |
| GET    | `/admin/owners/{uuid}`            | Admin     | —                                       | Получить владельца (полные данные) |
| GET    | `/admin/owners/public/{uuid}`     | User+     | —                                       | Получить владельца (публичные данные) |
| PATCH  | `/admin/owners/{uuid}`            | Admin     | JSON: `{ "name", "description" }`       | Обновить владельца             |
| DELETE | `/admin/owners/{uuid}`            | Admin     | —                                       | Удалить владельца              |

### Сообщения (messages)

| Метод | URL          | Доступ | Тело / параметры                                              | Описание                      |
|-------|--------------|--------|---------------------------------------------------------------|-------------------------------|
| POST  | `/messages`  | User+  | JSON: `{ "owner", "author", "text" }`                         | Создать сообщение             |
| GET   | `/messages`  | User+  | query: `page`, `page_size`, `status` (`new`/`read`), `owner`  | Список сообщений с фильтрами  |
